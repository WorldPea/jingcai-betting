# 竞彩投注全流程闭环

## Overview

赛前分析 → 投注记录 → 赛果通知 → 结算 → 复盘 → 优化沉淀的闭环。核心原则：**所有判断基于去水概率和多源情报交叉验证；所有优化必须沉淀（先进记忆，成熟后合并进本 skill）**。

## 数据与状态

| 文件 | 内容 |
|---|---|
| `~/.claude/jingcai/ledger.md` | 投注台账（唯一事实来源：下单赔率、投入、状态、收盘赔率、回款、累计统计）。**首次使用自建空文件** |
| `~/.claude/jingcai/reviews/YYYY-MM-DD.md` | 每期复盘记录 |
| `references/method.md` | 分析管线参数与工具通路（本 skill 内置，**开工先读全文**） |
| `references/format.md` | 方案输出格式硬性要求（整十金额、命中概率列、梯度等，本 skill 内置） |
| memory: 当期待跟进状态 | 使用者自建 memory 跟踪本期待决策/已投注（动态演进） |

## 阶段流程

### 1. 赛前分析（触发：截图/场次编号/队名/"出方案"）

**开工硬动作（不可省略，2026-06-14 用户确立）：** 收到分析请求后，**第一个动作**是 `Read` 本 skill 的 `references/method.md` 全文，按其第 1-5 步（含第 3 步 a-f 全部子项）建 TodoWrite 逐项打勾执行。**禁止凭下方概述的一句话管线走、禁止用 WebSearch 摘要替代整个第 3 步**；任何子步骤要跳过须显式向用户说明理由，不得静默省略（曾连续漏国际比价 + 泊松反推被纠正）。第 3 步必须展开：a) 国际比价（Pinnacle/the-odds-api）→ b) 双法去水保守取值 → c) 时点对齐 → d) 触发门 → f) **比分级泊松反推交叉验证**（反推 λ 找盘口间不自洽；比分推荐选号亦用此分布排序）。

**多源交叉论证硬动作（2026-06-25 用户确立，每个方案不可省略，绝不只靠 Pinnacle 单家）：** 出方案前必须跨 ≥3 类信息源交叉论证：
- ① **多家盘口共识**——the-odds-api 拉欧英美三区（`regions=eu,uk,us`）数十家（实测约 48 家）h2h+spreads+totals，python 算各家去水概率的**均值 + 离散度 σ + 让球/大小线分布**：σ 小=市场高度确信、σ 大=不确定（方案须标注）、某家明显偏离均值=潜在错价线索（再多源核验）。
- ② **预测市场**——`sports-skills markets match_markets --sport=worldcup` 取 Kalshi+Polymarket 群体真金概率，与庄家定价双源校验（中小球队场常未挂盘/流动性低，仅热门场有；缺失则显式注明"预测市场无盘、降级"）。
- ③ **情报源**——WebSearch / `sports-skills news fetch_items` 拉首发/伤停/动机/Opta 预测（⚠公开情报已被盘口定价，**只用于比分选号 + 首发公布后1-2h滞后窗口 + 轮换/稳腿风险标注**，绝不据此逆市场下注，见 method 第15条）。

每场方案须呈现"**共识方向 + 离散度(确信/分歧) + 情报修正点**"；任一源缺失时显式声明降级，不静默省略。

**🚦 出方案前强制自检清单（2026-06-26 用户确立·交付前逐项确认，别靠用户当人肉 QA）：** 本轮 6 次纠正中绝大多数是"规则已存在、第一版没执行"，故出方案/比分串前必须逐条过：
- [ ] **数据源**：用户给了低水盘口（尤其波胆）吗？→ 直接去水用之，没舍近求远用模型反推（门禁3）
- [ ] **比分主推自洽**：每场"方向措辞 vs 主推比分"一致吗？说大胜就给大比分，**没拿泊松/DC 峰值当主推**（门禁2）
- [ ] **大胜方向场**：复式覆盖了大比分范围（0:2/0:3/0:4）吗？没单押一个偏小比分
- [ ] **零封腿**：押 0:X 的配了"弱队进1球"对冲号（1:3/1:2）吗（门禁5）
- [ ] **情报**：≥2 个独立信源吗（同源转载算1个）？**没据未验证情报逆市场改概率**（门禁1）
- [ ] **比分串模板**：按 [[sporttery-betting-method]] 门禁 + 标准模板5步（选m场→递减投注→盈亏全景三件套）吗？没即兴（门禁4）
- [ ] **复式**：主动给了"中2场实际净赚多少"（废注稀释效应）吗？复式高命中 ≠ 中了赚得多
- [ ] **用户指定主力档位**（如"2串1作主力"）：保留了其他档位、只调权重，没删掉吗
- [ ] **真金投注**：第一版经得起逐条质疑吗？涉及真金别迭代式挤牙膏
> 门禁全文见 [[sporttery-betting-method]] 决策规则区「🚫 门禁」。

五步管线概览：Chrome/JSON 直采体彩赔率 → python 去水 → **国际比价 + 泊松反推**找错价 → 三源情报校验（析页伤停轮换/固定奖金页赔率变动/WebSearch首发+Opta）→ 按 `references/format.md` 出方案（整十金额、命中概率列、结果分布、稳健/平衡/博大梯度）。

**每期方案末尾额外附"比分推荐"独立板块（2026-06-13 用户确立）：** 每场主推1 + 备选2（共3注），用户**每注 50 元**自行额外购买。金额与盈亏**独立于 10000 资金池**（不占娱乐仓预算、不进累计盈亏）；写入 ledger"比分推荐台账"只追命中率（主推/前三/方向三档）。依据=比分盘 had 赔率 + 情报（半场比分/进攻数据）。诚实提醒：比分是 ~40% 水位的最高随机彩票，纯娱乐。

### 2. 投注记录（触发：用户说"已投/就买这个/确认"）

**方案≠投注，只记用户明确确认的单。** 写入 ledger.md：内容、下单赔率、投入、状态=待开。提醒用户：开赛前若赔率大幅变动会影响判断，可随时让我复核。

### 3. 赛果与结算通知（自动，可选）

可选自动化：定时任务（macOS launchd / Linux cron）→ `scripts/daily-jc-saiguo.sh`：抓赛果开奖页 → 按 ledger 逐腿结算 → 飞书 bot 发摘要 → 回写 ledger（状态/回款/累计统计）。**需先按 plugin README 配置环境变量（飞书 bot id、可选代理、claude 可执行路径）**，未配置则跳过此自动化。手动触发随时可用：用户说"看下赛果"。

### 4. 复盘（触发：一期全部结算后，或用户说"复盘"）

写 `reviews/YYYY-MM-DD.md`，固定四节：
1. **结果**：每单输赢、本期盈亏、累计盈亏
2. **逐腿判断质量**：预测概率 vs 实际结果；**CLV 检验**——下单赔率 vs 收盘赔率（固定奖金页有完整时间序列），下单价高于收盘价=判断领先市场
3. **信息源有效性**：本期哪些信号（轮换/伤停/赔率变动/国际比价）指对了、哪些误导了
4. **优化点**：具体可执行的改进（信息源、参数、仓位结构），每条标注【待验证】

### 5. 优化沉淀（复盘后立即执行）

- 新发现 → 更新对应 memory 文件（管线参数、工具坑、决策规则）
- 连续 2 期以上验证有效的优化 → 合并进本 SKILL.md（修改前先想清楚改哪一节，外科手术式修改）
- 被证伪的旧结论 → 从 memory 删除，不留尸体

## 比分串方案标准输出模板 v2（2026-06-25 立·2026-06-26 升级·⚠强制每次按此输出，新会话同样照此）

用户核心诉求＝**以小搏大、比分及比分串是绝对重点、命中率优先**（见 [[betting-user-habits]]）。出比分方案一律按下面流程 + 格式输出（主盘方案仍按 format 完整附上、作配角）。
**交付物固定 6 件（缺一不可）：① 各场去水表（主推+备选+概率+赔率）② 剔场+基底 ③ 单关/2串1 排序 ④ 推荐组合 ⑤ 完整逐注票面表（真实队名、可直接照投，用户 2026-06-26 明确"不要自己组合"）⑥ 盈亏全景三件套。全程过 [[sporttery-betting-method]] 门禁 + 上方「出方案前强制自检清单」。**

**第⓪步 数据源 + 剔场（v2 新增·先做）：**
- **数据源（门禁3）**：用户给了低水盘口波胆截图 → **直接对每场全部比分赔率去水＝市场隐含比分概率**（比泊松/DC 准、含全部情报）；无波胆数据才用 λ差反推 DC 兜底。
- **剔场（method 第21条·分散度信号）**：每场去水后看前 4 档比分——**有清晰主导比分（最高档明显高）＝方向干净、做基底；前 4 档挤在 ≤2pp 内无主导 / 双无动机场 ＝方向乱、踢出基底只观察**。实证：06-26 剔厄/土（果然爆冷乱战）、06-27 剔挪法（1.6pp）。

**第①步 各场比分分布 + 命中率**——优先用第⓪步市场去水概率（无波胆才用 λ差反推 DC）。输出表：`场次 | 主推 | 命中% | 备选(相近比分) | 赔率`。**主推必与方向措辞自洽（门禁2，禁泊松/DC 峰值乱推）。**
> ⚠️ **不再做"悬殊度分级选基底"（2026-06-25 弃用，见 method 第19条）**：曾用 λ差大小判断"哪场押比分有底/该选作基底"，连续两期反向证伪（06-25 精选可测带方向 0/3、悬殊场 2/2），该钩子无理论根基+无数据+指向反向，已退场。**λ差只用于反推 λ 算分布，不用于选腿。** 保留认知："比分=~15%乐透、任何场押单一比分都低命中"，不预设某类场"更有底"。

**第②步 选定 m 个基底场**（纯按模型命中率排序）：
- 按 DC 命中率从高到低选 m 场做基底（不再做可测性筛选）；诚实标注"命中率最高 ≠ 押得中，全是低命中乐透"。
- m 推荐：**m=3 精准狙击**（成本低、盈亏点低至中1场）／ **m=5 覆盖宽**（容错高、盈亏点中2场）。
- 每场主推比分＝市场去水/DC 峰值；要复式时纳入该场命中率 ≥10% 的全部相近比分（注意复式只覆盖峰值附近、防不住方向性爆冷/血洗）。
- **鱼腩零封必配对冲（门禁5）**：押 0:X（弱队零封）的场，复式必含一个"弱队进1球"对冲号（如比利时打新西兰 0:2/0:3 必配 1:2；塞内加尔打伊拉克 2:0 配 2:1），防鱼腩冷不丁进球暗杀（06-26 突尼斯 1:3 中招、累计第 N 次）。

**第③步 投注结构（二选一）+ 票面**：
- ①**递减投注**：钱跟命中率走，单关投最多、关数越高投越少（典型每注比例 单关:2串:3串…≈6:4:2…，金额取偶数=整倍数）。分票输出：`票1单关(m注) / 票2·2串1(C(m,2)注) / 票3·3串1(C(m,3)注)…`，每行 `组合|赔率|投注|命中回`。⚠ 选定 m 场全覆盖 → 自动 a=C(m,2),b=C(m,3)… 一致无漏洞；**不要各档独立选前N个**（会出"中了却没买对应低关串"的覆盖漏洞）。
- ②**2串1 主力 + 3/4串博大**（用户 2026-06-26 偏好的以小搏大结构，已实操）：基底场两两 2串1 复式占大头（含对冲号），3串1/4串1 各取概率最高 5 个小额博大；4串1 需扩 1 场（取剔场外命中率最高的胶着场作第4腿）。
- **不论哪种结构，本步必出完整逐注票面表**（交付物⑤：真实队名、每注 `组合|赔率|投注|命中回`，用户照着打、不自己组合）。

**第④步 盈亏全景**（必出）：输出表 `命中场数 | 概率 | 总回报 | 盈亏`，并标注 **盈亏平衡点**（中几场回本）、**期望回收率**、**不亏概率**。递减投注把盈亏点压低（m=3 常中1场即回本）；剔除弱场提高期望回收（剔波黑类 −19% 弱腿，回收率 82%→88%+）。

**第⑤步 博大附加**（可选，"简单投点"）：4/5/6串1 需扩场（基底+额外场），命中率 <0.05% 纯彩票 → 建议每注小额（如10元），诚实标注约 99.9% 打水漂、且必纳入悬殊场。

**诚实收尾**（固定一句）：所有比分串负 EV、大概率亏、是"买希望"非赚钱手段；当输得起的钱玩、别加码、别重仓。

> 全部金额可按总预算等比缩放（命中率、盈亏点不变）。单关需该场比分支持单关（crs 的 bettingSingle），不支持则去掉单关、盈亏点退回中2场。

## 硬约束（任何阶段不可违反）

- 组合 EV 直接代入去水输出数组，禁止手抄中间值
- 只用低水盘做腿（胜平负/让球 ~12.9%）；比分串≤预算15%；二串是性价比上限；可单关则优先单关
- 二串两腿命中概率都须够强（去水 ≥~58%）；双结果/中概率腿（让1负、不败等去水仅 ~50-64%，直觉稳实则弱）不与强腿混串——拆单关或弃用（连续2期翻车：06-12 韩让1负64%、06-13 巴不败~50% 串死整票）
- 疑似 +EV 冷门必须多家书商交叉验证（favourite-longshot bias）
- 推荐金额全部整十；每方案必带命中概率
- 诚实告知负 EV 与娱乐预算属性，一句即可不说教
- 台账只记用户确认的投注，绝不替用户决定"已投"

## Common Mistakes

| 错误 | 纠正 |
|---|---|
| 把"出过方案"当成"已投注"写台账 | 等用户明确确认 |
| 复盘只看输赢不看 CLV | 短期输赢是噪声，CLV 才是判断质量的信号 |
| 优化点直接改 skill | 先进 memory 标【待验证】，2 期验证后才合并 |
| 赛果通知漏更新累计统计 | 每次结算后重算累计表 |
| 凭 skill 概述走、漏第 3 步比价/泊松 | 开工先 Read 完整 `references/method.md` + 建 TodoWrite 逐项执行，第 3 步展开 a-f |
