# 竞彩赛前分析（极简版）

## Overview

只做一件事：赛前**多源分析出方案**。读用户提供的赔率（截图/文字）→ 去水算公平概率 → 国际比价 + 泊松反推找错价 → 情报校验 → 出方案（含命中概率、梯度、比分串）。核心原则：所有判断基于去水概率和多源情报交叉验证；负 EV 如实告知，纯娱乐。

> **赔率一律由用户提供（截图或赔率文字），本工具不抓取任何彩票官网。** 本精简版也**不含**投注台账、赛果通知、复盘等运营闭环——只是个"看盘出方案"工具。

## 分析流程

**开工硬动作（不可省略）：** 收到分析请求后，**第一个动作**是 `Read` 本 skill 的 `references/method.md` 全文，按其五步建 TodoWrite 逐项打勾执行。禁止凭概述的一句话管线走、禁止用 WebSearch 摘要替代整个比价+泊松步骤；任何子步骤要跳过须显式说明理由。

**多源交叉论证（每个方案不可省略，绝不只靠单家）：** 出方案前跨 ≥3 类信息源交叉论证：
- ① **多家盘口共识**——the-odds-api 拉欧英美三区数十家 h2h+spreads+totals，算各家去水概率的均值 + 离散度 σ + 让球/大小线分布：σ 小=市场确信、σ 大=不确定（须标注）、某家偏离均值=潜在错价线索。
- ② **预测市场**——Kalshi+Polymarket 群体真金概率，与庄家定价双源校验（中小球队常无盘，缺失则标注降级）。
- ③ **情报源**——WebSearch 拉首发/伤停/动机/Opta（⚠公开情报已被盘口定价，只用于比分选号 + 风险标注，绝不逆市场下注）。

每场方案须呈现"**共识方向 + 离散度(确信/分歧) + 情报修正点**"；任一源缺失显式声明降级。

**🚦 出方案前强制自检（交付前逐项确认，别靠用户当人肉 QA）：**
- [ ] **数据源**：有低水波胆截图吗？→ 直接去水用之，别舍近求远用模型反推
- [ ] **比分主推自洽**：方向措辞 vs 主推比分一致吗？说大胜就给大比分，**没拿泊松/DC 峰值当主推**
- [ ] **零封腿**：押 0:X 的配了"弱队进1球"对冲号吗（1:3/1:2）
- [ ] **情报**：≥2 个独立信源吗（同源转载算1个）？**没据未验证情报逆市场改概率**
- [ ] **市场分散剔场**：波胆去水前 4 档挤在 ≤2pp 内无主导=方向乱、踢出基底只观察

五步管线概览：读用户赔率截图/文字 → python 去水 → **国际比价 + 泊松反推**找错价 → 公开情报校验 → 按 `references/format.md` 出方案（整十金额、命中概率列、结果分布、稳健/平衡/博大梯度）。

**每期方案末尾额外附"比分推荐"独立板块：** 每场主推1 + 备选2（共3注），每注 50 元自行额外购买。依据=比分盘去水概率 + 情报。诚实提醒：比分是 ~40% 水位的最高随机彩票，纯娱乐。

## 比分串方案标准输出模板

用户核心诉求＝**以小搏大、比分及比分串是重点、命中率优先**。出比分方案一律按下面流程输出（主盘方案仍按 `references/format.md` 完整附上、作配角）。交付物固定：① 各场去水表（主推+备选+概率+赔率）② 剔场+基底 ③ 单关/2串1 排序 ④ 推荐组合 ⑤ **完整逐注票面表**（真实队名、可直接照投）⑥ 盈亏全景。

**第⓪步 数据源 + 剔场：**
- **数据源**：有低水波胆截图 → 直接对每场全部比分赔率去水＝市场隐含比分概率（比泊松准、含全部情报）；无波胆才用 λ差反推 DC 兜底。
- **剔场**：每场去水后看前 4 档比分——**有清晰主导比分（最高档明显高）=方向干净做基底；前 4 档挤 ≤2pp 无主导 / 双无动机场 =方向乱踢出基底只观察**。

**第①步 各场比分分布 + 命中率**——输出表：`场次 | 主推 | 命中% | 备选(相近比分) | 赔率`。主推必与方向措辞自洽。

**第②步 选 m 个基底场**（按 DC 命中率排序）：m=3 精准狙击 / m=5 覆盖宽。每场主推=市场去水/DC 峰值；要复式时纳入命中率 ≥10% 的相近比分。**押 0:X（弱队零封）必配一个"弱队进1球"对冲号**（如 0:2/0:3 配 1:2）。

**第③步 投注结构 + 票面**：①**递减投注**（钱跟命中率走、关数越高投越少；选定 m 场全覆盖自动 C(m,2)/C(m,3)… 无漏洞）或 ②**2串1 主力 + 3/4串博大**。混合过关「M场全关数复式」抗单场翻车（挂1场剩低关组合照中）。**本步必出完整逐注票面表**（真实队名、每注 `组合|赔率|投注|命中回`，用户照打不自己组合）。

**第④步 盈亏全景**（必出）：表 `命中场数 | 概率 | 总回报 | 盈亏`，标注盈亏平衡点（中几场回本）、期望回收率、不亏概率。

**诚实收尾（固定一句）：** 所有比分串负 EV、大概率亏、是"买希望"非赚钱手段；当输得起的钱玩、别加码、别重仓。

> 全部金额可按总预算等比缩放（命中率、盈亏点不变）。该玩法支持单关时优先单关（损耗减半）。

## 硬约束（任何阶段不可违反）

- 只用低水盘做腿（胜平负/让球 ~12.9%）；比分串≤预算 15%；二串是性价比上限；可单关优先单关
- 二串两腿命中概率都须够强（去水 ≥~58%）；双结果/中概率腿（让1负、不败等去水仅 ~50-64%）不与强腿混串——拆单关或弃用
- 疑似 +EV 冷门必须多家书商交叉验证（favourite-longshot bias）
- 推荐金额全部整十；每方案必带命中概率
- 诚实告知负 EV 与娱乐预算属性，一句即可不说教

## Common Mistakes

| 错误 | 纠正 |
|---|---|
| 凭 skill 概述走、漏比价/泊松 | 开工先 Read 完整 `references/method.md` + 建 TodoWrite 逐项执行 |
| 用泊松/DC 峰值直接当比分主推 | 主推从攻防质量/对手防守/情报出发，与方向措辞自洽 |
| 押 0:X 零封不配对冲号 | 必配"弱队进1球"号（1:3/1:2），防鱼腩冷不丁进球 |
| 据未验证公开情报逆市场调概率 | 公开情报已被盘口定价，只用于比分选号 + 风险标注 |
