**第3步 国际比价找错价票 + 优势验证门（2026-06-12 升级，Kelly 触发的唯一合法入口）**
a) **基准源优先 Pinnacle 直采（2026-06-12 实测可用）**：WebSearch 搜 `"队名 vs 队名 odds Pinnacle"` 拿到 pinnacle.com 比赛页直链 → Chrome 渲染（new_page → wait_for "Money Line"）→ Money Line – Match 三项即 1X2（美式格式，水位仅 ~3.6%，全球最准市场基准）。**第二/三独立源 Kalshi+Polymarket 一键齐取**：`sports-skills markets match_markets --sport=worldcup`（自动配对两平台同场盘口含价格，2026-06-12 实测两家价完全一致可互为校验；单平台通路见工具表）——与 Pinnacle 属不同性质来源（庄家定价 vs 群体共识），双源同向才算确认。备选：bet365/DraftKings 经 WebSearch（水位 ~6%）。
b) **双方法去水，保守取值**：同一组国际赔率分别跑按比例法和 power 法（二分求 k 使 Σp^k=1，fair=p^k，压冷门抬热门），**EV 取两法中较低者**。实测案例：波黑@4.90 按比例 -1.6% 看似近公允，power 法 -3.7%——冷门偏差陷阱实锤，此规则就是防它。
c) **时点对齐**：体彩侧必须用固定奖金页最后一行实时价（非截图旧价），与国际报价时间差 ≤1 小时，否则比价无效。
d) **触发门槛（全部满足才触发 Kelly）**：保守 EV ≥ +3%（0~3% 视为噪声不动作）且 ≥2 个独立来源（Pinnacle 算一个）确认同向 +EV。
e) **信号记账**：每次触发/未触发的边缘案例记入台账备注，赛后用 CLV 验证，累计 ≥10 样本后校准 +3% 阈值。
f) **比分级泊松反推交叉验证（2026-06-14 加入，找盘口间不自洽）**——第二/三验证维度，不替代 a) 的 1X2 比价：
   - **输入用真金基准**（Pinnacle/Kalshi/Polymarket 的让球主盘线 L_让 与大小主盘线 L_大；主盘=两边赔率最接近 1.9 那条；**禁用体彩自身赔率**，其 12.9%~40% 高水会污染 λ）
   - **反推预期进球**：主队 H=(L_大+L_让)/2、客队 A=(L_大−L_让)/2（让球以"主队让为正"取号；例 巴西让0.75+大小2.25 → H=1.5/A=0.75）
   - **泊松还原比分分布** P(h,a)=Pois(H,h)·Pois(A,a)，h,a∈0..6（python：`math.exp(-λ)*λ**k/math.factorial(k)`）
   - **聚合回体彩各玩法真实概率**：1X2（主胜 Σ_{h>a} / 平 Σ_{h=a} / 客胜 Σ_{h<a}）、让球（按体彩固定让球数聚合）、总进球（按 h+a 分档）、比分（直接 P(h,a)）
   - **逐项 EV=真实概率×体彩赔率−1 找偏离**；**盘口间不自洽本身是信号**：若泊松反推的 1X2 与 a) 直接去水的国际 1X2 差 >3pp，说明体彩某盘没跟齐（让球/大小盘 vs 1X2 盘有张力）
   - ⚠️**校准**：纯泊松低估低分平局，用 Dixon-Coles τ 修正（抬高 0:0/1:1、压低 1:0/0:1，ρ≈0.10）；强攻/强守球队 λ 按基本面微调；**比分盘 40% 水位下即便相对便宜仍多为负 EV → 比分反推主要用于「比分推荐板块选号排序」+「交叉验证方向」，不作正 EV 来源**
   - ⚠️**实测铁律（2026-06-14 首测得出）**：反推会吐出一堆"正EV"（如巴西vs摩洛哥跑出 0:2 +12.6%、0:0 +5.4%），但**冷门/低频比分的绝对正EV几乎都是模型噪声 + favourite-longshot bias 陷阱**（小概率比分 EV 对 λ 极敏感，λ 差 0.1 就翻盘）——**只信同方向比分间的相对排序**（该例真信号＝巴西赢的剧本里 1:0(+1.2%) > 2:1(深负)，因体彩把大众爱买的 2:1 压贵、1:0 相对慷慨），**绝不照搬绝对 EV 去追冷门比分**（违反"疑似+EV冷门必多源验证"硬约束）；且相对最优的 1:0 经 Dixon-Coles 修正后 EV 大概率转负→定位是"选相对最不亏的号"，非"挖正EV"
   - ⚠️**split/整球让球的泊松 EV 尤其不可信（2026-06-21 多次实证）**：体彩单方面加深让球（如德让 -1→-1.25、国际仍 -1.0）时，泊松对 split(-X/X.5)/整球盘常吐出 +20% 级**假正 EV**（德科科+1.25 +23.5%、比-1.25 +23.5%、新+0.75 +23.5% 全是假象）。双重根因：①**split 盘 EV 对"半输/半赢"方向极敏感**，公式写反就翻盘（净胜=N 时到底半赢还是半输要按 -X/X.5 拆开两腿逐一判定）；②**泊松系统性高估弱队不败**（DC 修正抬高低分平局）。**铁律：让球/大小一律用国际同线去水算 EV（同线最多差个水位、可靠）；仅跨线才动泊松，且只信方向不信绝对值**。
   - **最佳猎物窗口**：首发/伤停公布后 1-2h，国际盘已调 λ 而体彩让球/比分盘滞后未跟——此时泊松反推暴露的偏离最可能是真错价（呼应 d) 时点对齐）
   - **数据源补全 + λ 可信度降级链（2026-06-14 实证固化，解决"A/B 层常不齐"）**：
     · **B 层补全**——任何三向价（Kalshi/Opta/Polymarket/体彩去水皆可）都能用泊松**拟合反推 λ**：网格搜 H,A∈[0,4] 步0.02，最小化与目标 1X2 概率的平方差。实证：纯 1X2 拟合 H1.62/A0.78 vs 让球+大小直接反推 H1.50/A0.75，Δ<0.2 高度一致 → **λ 可用源从"几乎只剩 Pinnacle 亚盘"扩到"任何有三向价的源"**，不齐全痛点大幅缓解
     · **多源融合**——A层直接亚盘反推 + B层各三向拟合 + C层 xG，取中位数/加权；单源缺失其他补位
     · **可信度降级链**——高（≥2 源 λ 一致 Δ<0.2）/ 中（1 源直接亚盘 或 2 源三向拟合）/ 低（单源）。**低可信场次方案必须标注"λ 仅供参考"、降权或直接跳过，绝不硬给精确数字**
     · **修不了的诚实边界**——世界杯赛季初 xG 稀缺、新阵容、冷门小联赛流动性低导致的 λ 不准，是数据本身不存在，非通路问题，只能靠降级链标注、不假装精确
旧案例留档：墨西哥vs南非，体彩墨胜 1.30 偏贵（EV -17%）✓被门挡住；南非 8.40"疑似+EV"未经双法+多源验证，属未达门槛的观察信号。

## 历史沉淀的规则与教训（第1-3期 2026-06-12~14，已压缩）

- ✅ **bankroll=10000**（娱乐仓/狙击仓分仓见第2步）。
- ✅ **双结果腿≠稳腿（已并入 SKILL 硬约束）**：让1负/不败等中概率腿（去水仅 ~50-64%）连续2期串死整票，二串两腿都须去水 ≥65% 才用，否则拆单关/弃用。

### 方法论背景（2026-06-25 补·定位参考，非操作步骤）

足球预测主流方法论体系，本管线在其中的位置：①**比分级**：泊松/Dixon-Coles（在用）、Bivariate Poisson、贝叶斯层级（Baio-Blangiardo）；②**结果级**：Elo 评分（FIFA 排名 / 538 SPI）、ordered logit、ML；③**强度输入**：xG（Opta/Understat）、市场赔率反推（在用）；④**赛事模拟**：蒙特卡洛（Opta supercomputer / 538 冠军预测）；⑤**元方法**：**市场赔率本身＝最准单一预测器**（学术反复证明 Pinnacle 收盘价 / Betfair 交易所价胜过几乎所有公开模型——有效市场，所有信息已被定价）、集成 ensemble。**本管线＝「市场反推 λ(supremacy/total) → DC 比分分布」＝用最准输入喂经典比分模型，主流且靠谱**。⚠️**天花板**：所有方法对"具体比分"的预测力都极有限（比分高方差、最可能比分也才 ~15% 命中）——方法论先进只改善胜平负/让球/大小的**区间**精度，救不了押单一比分的低命中。**操作含义：主盘信市场赔率(基准)、比分当乐透。**

## 决策规则（方案构建时硬约束）

### 🚫 门禁（2026-06-26 立·绝对禁止·均为本人实际越线被用户当场纠正的教训）

1. **多源验证门禁**：情报必须 ≥2 个**独立**信源（同一条新闻的多家转载＝1 个源，不算）；**严禁据未充分验证的公开情报逆市场调整概率/胜率**（公开情报只用于比分选号 + 风险标注，见第15条）。越线例：单凭一条日媒转载把日本胜率 50%→40% 去对赌市场。
2. **比分主推自洽门禁**：出每场比分主推前，**必须核对"方向措辞 vs 主推比分一致"**（说大胜就给大比分、说低进球就给小比分）；**严禁用泊松/DC 峰值直接当主推**（见第10条）。越线例：判"德大胜"却推 0:1（德赢1球）。
3. **数据源优先门禁**：用户已提供低水盘口（尤其波胆）时，**必须直接去水用之**，严禁舍近求远用模型反推算概率（见第20条）。越线例：用户给了国际盘波胆，我还在用 the-odds-api 反推 λ。
4. **模板纪律门禁**：出比分串方案**必须先 Read skill 标准模板、严格按5步**（选m场→递减投注→盈亏全景三件套），严禁凭记忆即兴。越线例：第一版给"统一注额全关复式"而非递减投注。
5. **零封对冲门禁**（操作级）：押单方零封比分（0:X）**必须配一个"弱队进1球"对冲号**（0→1，如 1:3/1:2），否则不押该腿（见第22条）。已第 N 次被弱队进球暗杀（突尼斯 1:3）。

> 元教训：本轮 6 次纠正中绝大多数是"规则已存在、第一版没执行"——核心不是缺知识，是缺开工前自检。出方案前**强制走一遍上述门禁 + [[jingcai-betting-loop]] 开工自检清单**，别靠用户当人肉 QA。真金投注时第一版就要经得起逐条质疑。

- **损耗控制三原则（2026-06-12 与用户确认，娱乐仓选票同样适用）：** ①无票跳过（省100%的水，负EV游戏里不下注就是最优解）＞②能单关不串（省一半串关损耗，先查赛程总表单关标记：u-dan=可单关、u-cir=仅过关）＞③只用低水盘做腿。境外低水位平台（Pinnacle等）仅作比价基准，大陆用户经其投注违法，不推荐不协助
- **水位档位：** 胜平负/让球 ~12.9%；总进球/半全场 ~25.4%；比分 ~40-41%。只用低水盘做腿，比分串只当彩票（≤总预算15%）
- **损耗规律：** 低水盘每加一腿期望回收 ×0.886。二串 -21.5%、三串 -30%、四串 -38%。性价比只由腿数和盘口类型决定，**二串低水盘是性价比上限**
- **最优结构：** 三条强腿打"2串3轮换"（体彩原生支持）优于三串一——EV 相同但单腿翻车仍有一张单存活。用户已采纳过此结构
- **让球语义：** 让 N 负 = 让球后主队输（净胜 ≤ N-1 或不胜）
- 诚实告知所有方案 EV 为负、属娱乐预算，每轮提醒一句即可不说教
